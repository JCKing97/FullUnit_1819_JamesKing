Agent Mind as a Service API documentation version 0.1

http://localhost:8080/api/

Welcome to the documentation for the agent mind as a service API, the service hosts agents minds for a game-theoretic model that uses mixed indirect and direct reciprocity mechanisms. These agents have their own memory of events (input as percepts) which the agent then uses to form beliefs about their world upon which they can act.

/strategy

The strategies an agent can be.

/strategy get

get

Retrieve a JSON object containing names, descriptions and options of the strategies an agent can be. The strategies listed here aren't an exhaustive list.

/generation

A generation belongs to a community and agents reside in a generation.

/generation post

post

Create a new generation for an existing community in the service. If the return value of success is false, an error message will be returned in the message field of the JSON object.

post /generation

Create a new generation for an existing community in the service. If the return value of success is false, an error message will be returned in the message field of the JSON object.

/percept

Agents perceive events and happenings in an environment, input percepts to them.

/percept/action

A particular type of percept where an action has been taken and the agent has perceived it.

/percept/action/interaction post

An interaction between a donor-recipient pair where the donor has committed to an action.

post

Give an agent a percept of an interaction between a donor-recipient pair of agents at a specific timepoint and the action (defect or cooperate) that the donor took.

post /percept/action/interaction

Give an agent a percept of an interaction between a donor-recipient pair of agents at a specific timepoint and the action (defect or cooperate) that the donor took.

/action

An action is taken by an agent at a specific timepoint, this can either be idle, gossip or action.

/action get

get

Get the action that an agent commits to at a particular timepoint with the beliefs that have been built from perceptions. The action can either be idle, gossip (positive or negative, with about and recipient parameters) or action (cooperate or defect, with recipient parameters)

get /action

Get the action that an agent commits to at a particular timepoint with the beliefs that have been built from perceptions. The action can either be idle, gossip (positive or negative, with about and recipient parameters) or action (cooperate or defect, with recipient parameters)

  • Response

  • community: required (integer)

    The id for the community that the actor belongs to

  • generation: required (integer)

    The id for the generation that the actor belongs to

  • player: required (integer)

    The id for the actor
